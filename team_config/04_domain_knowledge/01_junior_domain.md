# Junior Domain Knowledge (TaskFlow)

**Cumulative**: Load this AFTER 00_saas_project_management.md

## What Juniors Learn (Sprint 1-5)

### Basic Workflows
- How to move cards through board
- Git integration basics (branch naming, PR links)
- Sprint ceremonies (planning, standup, retro)
- Common edge cases: "What if card has no points?"

### Customer Context
- Software teams use TaskFlow daily
- Real-time collaboration is critical
- Downtime = teams can't work = high impact

### Common Questions to Ask in Planning
- "What happens if user is offline?"
- "Do we validate input?"
- "What's the error message?"
- "Is this a breaking change?"
